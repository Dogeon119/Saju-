## 0. 브랜드 한 줄

**"감청빛 밤하늘 아래, 별빛 금으로 눌러 쓴 달빛의 감정서(鑑定書)."**
> (2026-07-22 月夜/月白 리브랜드 — 구 "흑단 위 금박"에서 전환. 현행 팔레트·개정은 **부록 I**가 최종.)

고급스러움은 색의 수가 아니라 절제에서 나온다. 월하의 화면은 명품 인쇄물의 문법을 따른다 —
낮은 채도, 어두운 명도 베이스, 금속성 액센트 한 색, 비대칭 조판, 각이 살아 있는 도장.

### 반(反)AI 원칙

이 시스템의 존재 이유는 "AI가 만든 티"의 제거다. AI풍의 정체는 색이 아니라 **문법**이다:
가운데 정렬 + 알약 뱃지 + 균등 라운딩 + 균등 패딩 + 그라데이션. 이 조합을 금지하고
인쇄물 문법(비대칭 · 크기 위계 · 세로쓰기 · 괘선 · 도장)으로 대체한다.

## 1. 색 — 팔레트는 이 토큰이 전부다

모든 색은 CSS 변수로만 사용한다. **토큰 외 hex는 어떤 파일에도 존재할 수 없다** (검증: §7).

> ⚠️ **현행 팔레트 값은 [부록 I](#부록-i--月夜月白-리브랜드-2026-07-22-밤하늘-방향-확정)(月夜/月白)가 최종이다.** 아래 §1 원본 hex와 부록 B(한지 기본)는 리브랜드 이전의 역사적 기록 — 토큰 **이름·역할**만 유효하고 값은 부록 I를 따른다.

```css
:root {
  /* 지면 (紙面) */
  --ink-0: #0E0D0C;        /* 페이지 최심부 — body 배경 */
  --ink-1: #131211;        /* 기본 배경 — 장면 지면 */
  --ink-2: #1C1A17;        /* 카드 · 상승면 */
  --ink-3: #262320;        /* hover · 활성 상승면 */

  /* 괘선 (罫線) */
  --line: #2E2A24;         /* 기본 구분선 · 세로 괘선 */
  --line-strong: #3C372F;  /* 강조 구분선 · 입력 테두리 */

  /* 금박 (金箔) — 유일한 액센트 */
  --gold: #C9B37E;         /* 제목 강조 · 활성 탭 · 주요 버튼 */
  --gold-dim: #8C7C58;     /* 비활성 금 · 방문한 챕터 표시 */

  /* 한지 (韓紙) — 글자 */
  --paper: #E9E2D3;        /* 본문 */
  --paper-dim: #8F8A80;    /* 캡션 · 보조 · 세로 챕터 표기 */

  /* 낙관 (落款) — 도장 전용 */
  --nakgwan: #8F3B32;      /* 도장 테두리 */
  --nakgwan-text: #B0574C; /* 도장 글자 · 경고 문구 */

  /* 오행 5색 — 흑단 위 채도 억제판 */
  --e-mok:  #7FA07A;  /* 木 */
  --e-hwa:  #C06B5A;  /* 火 */
  --e-to:   #C2A368;  /* 土 */
  --e-geum: #B9BCB6;  /* 金 */
  --e-su:   #7392A8;  /* 水 */
}
```

### 사용 규율

| 색 | 역할 | 절제 규칙 |
|---|---|---|
| `--gold` | 유일한 액센트 | 한 화면에서 제목 강조 + 활성 표시 정도. 본문에 금색 금지 |
| `--nakgwan` 계열 | 낙관 도장 전용 | **화면당 도장 1개 이하.** 붉은색이 흔해지는 순간 격이 죽는다 |
| 오행 5색 | 사주판 · 오행 바 · 오행 칩 전용 | UI 장식에 유용(流用) 금지. 배경 틴트로 쓸 땐 22% 투명도(`#7FA07A22` 형식이 아니라 `color-mix` 또는 opacity로) |
| `--ink-*` | 명도 위계 | 0→3 순서 준수. 단계 건너뛴 임의 명도 생성 금지 |

**금지:** 팔레트 외 색 · 2색 이상 그라데이션 · 네온 · 보라 계열 · 순수 흰색(#FFF)과 순수 검정(#000).

## 2. 타이포그래피 — 2종으로 끝

| 용도 | 폰트 | 크기/행간 |
|---|---|---|
| 장면 제목 | 명조 (Noto Serif KR) | 24~26px / 1.35 — `--paper`, 부분 강조만 `--gold` |
| 본문 | 명조 (Noto Serif KR) | 14px / **1.9** — `--paper` |
| 캡션 · 세로 챕터 표기 | 명조 | 12~13px — `--paper-dim` |
| UI 컨트롤 (버튼·입력·탭) | 고딕 (시스템 스택) | 13~14px |

- Noto Serif KR은 `next/font/google`로 셀프호스팅 (weight 400·600만). CDN `<link>` 금지.
- **위계는 라벨이 아니라 크기로 만든다.** "핵심 한 줄 —", "💡 TIP" 같은 라벨 텍스트 금지.
- 행간 1.9는 본문의 격을 만드는 핵심값이다. 좁히지 말 것.
- 최소 크기 12px. 자간은 세로쓰기 챕터 표기에만 `letter-spacing: 7px`.

## 3. 조판 문법 (인쇄물 문법)

### 3.1 챕터 헤더 — 세로쓰기 괘선 기둥

챕터 표기는 본문 좌측의 세로 기둥이다. 가운데 정렬 타이틀 금지.

```
표기 형식: 제四장 · 그대의 인연
- 장 번호는 한자 숫자: 一 二 三 四 五 六 七 八 九 十
- "제"와 "장"은 한글. 서장은 "서장", 종장은 "종장" (번호 없음)
```

```jsx
<div className="chapter-rail">   {/* writing-mode: vertical-rl */}
  <span>제四장 · 그대의 인연</span>
  <div className="rail-line" />  {/* width 1px, background var(--line) */}
</div>
```

CSS 핵심: `writing-mode: vertical-rl; letter-spacing: 7px; color: var(--paper-dim); font-family: 명조`.
기둥 아래로 1px `--line` 괘선이 본문 높이만큼 이어진다.

### 3.2 장면 레이아웃 — 2단 비대칭

```
┌─────────┬──────────────────────────────┐
│ 세로기둥 │  장면 제목 (26px 명조)         │
│ 제四장   │                              │
│  ·      │  본문 문단 (14px / 1.9)        │
│ 부제    │                              │
│  │      │  ──────────────── (괘선)      │
│  │      │  당부 한 줄            [緣]도장 │
└─────────┴──────────────────────────────┘
```

- 가운데 정렬은 도장·인장 요소 내부에만 허용.
- 좌측 기둥 폭은 콘텐츠 자연폭, 본문과의 간격 16px.

### 3.3 낙관 도장 (경고 · 당부 · 핵심 표식)

알약(pill) 뱃지의 전면 대체물.

```jsx
<div className="nakgwan">緣</div>
/* width/height 34px · border 1.5px solid var(--nakgwan)
   border-radius: 0 (절대 각) · 글자 15px 명조 var(--nakgwan-text) */
```

- 한자 1자만: 緣(인연) 財(재물) 戒(경계) 吉(길함) 命(운명) 情(마음) 등 문맥에 맞게.
- 도장 옆 문구는 `--paper-dim` 12.5px, 괘선(`border-top: 1px var(--line)`) 위 행에 배치.
- 화면당 1개 이하 (§1 규율).

### 3.4 라운딩 규율

| 요소 | 라운딩 |
|---|---|
| 카드 · 장면 지면 | 8~10px |
| 도장 · 괘선 · 구분선 · 사주판 글자 칸 | **0px** |
| 버튼 · 입력 | 6px |

그 사이의 임의 값, 그리고 `border-radius: 9999px`(pill) 금지.

## 4. 컴포넌트 사양

### 챕터 탭 (상단 고정 내비게이션)
- 배경 `--ink-0` 반투명 + blur, 하단 `--line` 1px.
- 탭은 사각(라운딩 6px 이하) — pill 금지. 활성: `--gold` 글자 + 하단 2px `--gold` 밑줄. 비활성: `--paper-dim`. 잠금(미생성): `--gold-dim` + 도트.

### 장면 카드 3유형
| 유형 | 지면 | 구성 |
|---|---|---|
| `data` (사주판·오행바) | `--ink-2` 카드 | 기존 PillarBoard류. 오행색은 §1 억제판 토큰으로 교체 |
| `story` (본문) | `--ink-1` 지면 위 직접 | §3.2 비대칭 조판. 카드로 감싸지 않는다 — 지면에 바로 조판 |
| `warn` (경고·당부) | `--ink-1` | 괘선 상단 + 문구 + 낙관 도장 (§3.3) |

### 버튼
- 주요("사주 펼치기" 등): `--gold` 배경 + `--ink-0` 글자. **화면당 1개.**
- 보조: 투명 배경 + `--line-strong` 테두리 + `--paper` 글자. hover 시 `--ink-3`.

### 입력 폼
- 배경 `--ink-2`, 테두리 1px `--line-strong`, 포커스 시 `--gold-dim`. 라벨 `--paper-dim` 12px.

## 부록 D — 관리자 콘솔 예외 구역 (2026-07-13, 사용자 요청)

`/admin`(및 `.admin-shell` 하위)은 **브랜드 규율의 예외 구역**이다 — 내부 도구는 모던 화이트 SaaS 문법을 따른다.

본문 §1의 토큰과 금박·낙관·명조를 쓰지 않는다. 순백 금지(§1)·이모지 금지(§6)는 이 구역에도 각각 예외·유지로 적용된다.
접근은 3중: `/?view=admin` 숨은 입구 → 4자리 번호(`ADMIN_PIN`) → 관리자 계정 로그인(`ADMIN_EMAILS` 서버 검증).

### 부록 D-2 — 콘솔 시각 언어 v2 (2026-07-12, "가독성·전문성" 피드백)

흰 지면은 유지하되 위계와 액센트를 더한다.

- **배경 이층화**: 쉘 배경 `#F5F6F8`, 카드·리스트 `#FFFFFF` + 미세 그림자(`0 1px 2px rgba(17,20,24,.05)`) — 지면과 표면을 분리한다.

## 부록 E — 모션 규율 (2026-07-12, "전체적으로 애니메이션" 요청)

움직임은 **지면에 스며드는 잉크**처럼 — 짧고, 낮고, 한 방향(아래→위)으로만.

- **진입**: `fadeUp`(8~14px 상승 + 페이드) 단일 어휘. 지속 0.25~0.5s, 이징 `cubic-bezier(.22,1,.36,1)`.
  적용면: `.page-head` · `.card` · `form` · `.mode-list` · 장면(기존 §3). 리스트·그리드는 `.stagger`로 자식당 50ms 지연.
- **반응**: 버튼류 `:active`에 `scale(.98)` press 피드백. hover는 배경·색 전환 0.15s (기존 §4 유지).
- **차트**: 막대는 `scaleX/scaleY` 성장(0.5~0.7s, transform-origin 고정). 수치 자체는 애니메이션하지 않는다.
- **오버레이**: 모달 = 딤 fadeIn 0.18s + 본체 riseIn 0.22s. 토스트 = 하단 슬라이드업 0.25s, 2.4s 후 자동 소멸.
- **아코디언**(회원 탭 정리 카드): `grid-template-rows 0fr↔1fr` 전환 0.3s + 셰브런 90° 회전.
- **금지**: bounce·spring·회전 장식, 0.6s 초과 진입, hover에서의 큰 위치 이동, 무한 반복(상태 점 pulse 제외).
- `prefers-reduced-motion: reduce`에서 전부 무효 — 기능은 애니메이션 없이도 완결되어야 한다.

## 부록 G — 모바일 압축 스케일 (2026-07-12, "모바일 난잡·가독성" 피드백)

부록 C의 큰 활자는 **풀이 본문**의 원칙이다. 모바일(≤640px) **UI 표면**은 한 눈에 들어오게 압축한다.

- **한국어 개행**: `body{word-break:keep-all; overflow-wrap:break-word}` — 단어 중간 꺾임 금지 (전 화면 공통).
- **홈 기능 그리드**: 모드 리스트는 모바일에서 **2열 타일**(`.mode-grid`) — 도장 + 제목만, 설명·셰브런 숨김. 기능 전체가 첫 화면에 들어오는 것이 목적. 데스크톱은 행 리스트 유지.
- **유동 스케일 (v2, 2026-07-12 "화면 크기에 따라" 피드백)**: 고정 눈금 대신 **`clamp()`가 화면 폭을 따라 연속적으로** 조정한다 — 640px 절벽 금지. 컨테이너 패딩 `clamp(20,5vw,48)` · 히어로 제목 `clamp(27,7vw,46)` · 페이지 제목 `clamp(20,…,27)` · 도장 `clamp(45,…,58)` · 카드/폼 패딩 `clamp(16,2.6vw,24)` · 히어로 장식 `clamp(200px,44vw,350px)`. 미디어쿼리는 **구조 전환**(그리드·sticky 해제·행 조밀화)에만 쓴다. 홈 타일은 **가로형**(도장+제목 한 줄)이고 ≤380px 초소형 보정이 따로 있다.
- **위젯·리스트 행 설명은 1~2줄 클램프** 대신 짧은 원고를 우선한다 — 말줄임(…)은 최후 수단.
- 터치 타깃 44px 이상(부록 A)은 축소 대상에서 제외.

### 잘릴 수 없는 구조 원칙 (2026-07-12, 유저 전달 — 대기업 반응형 관행)

화면마다 고치지 말고 애초에 잘릴 수 없게 만든다.

1. **고정값 대신 상대값** — 폭은 `%·fr·max-width`, 크기는 clamp. `w-96`류 고정 폭 금지.
2. **모바일 퍼스트** — 최소 화면이 기본, 미디어쿼리는 확장 방향으로만.
3. **세로는 dvh** — `100vh` 금지(주소창만큼 아래가 잘린다). 전체 높이는 `100dvh`.
4. **Safe Area** — 하단 탭 `env(safe-area-inset-bottom)` + 본문 하단 `calc(84px + inset)` + 좌우 `max(패딩, inset-left/right)` (노치 가로모드).
5. **글자는 clamp()** — v2 유동 스케일이 이 원칙의 구현.
공유 카드(OG)는 1200×630 고정 비율 래스터라 기기와 무관하게 동일 — 추가 처리 불필요.

### H-2 제품 정체성 (ID) — 3초 룰

**ID-1** 3초 안에 "달빛 아래의 감정서"가 느껴져야 한다. **ID-2** 주인공은 감정의 언어(명조 표제·사주판·점수). **ID-3** 대시보드가 아니라 읽는 화면. **ID-4** 게스트 시선 순서 = 여기가 무엇을 봐주나 → 어떤 감정(모드) → 지금 시작. **ID-5** "3초 메시지를 강화하나, 그냥 있으면 좋은 정보인가?" 후자면 홈에서 뺀다.

### H-3 위계 3단 (HI) — 크기·명조·괘선으로만

**HERO**(화면당 1개·명조 대형·여백) → **PRIMARY**(핵심 문장·스탯 행·사주판) → **TERTIARY**(캡션·괘선 주석·배지, 저대비). **HI-1** HERO는 정확히 1개. **HI-2** 첫 스크롤 블록 최대 5~8개. **HI-3** HERO는 여백을 넉넉히. **HI-4** 위계는 크기·명조·괘선으로 — 그림자·브랜드 배경·굵기만으로 만들지 않는다(§6 box-shadow 금지와 일관).

### H-4 홈·탭 IA 청사진 (IA)

홈(月)은 **감정 도구 허브**: ① 히어로 인사(명조 대형 + 수묵 달) → ② 모드 5종 허브(행이 곧 상품) → ③ 게스트 유도/최근 감정서. **IA-1** 홈에 통계·대시보드 위젯을 쌓지 않는다(참고 지표는 오늘·달력·서재가 주인). **IA-2** 모드는 데스크톱 행 리스트/모바일 2열 가로 타일. **IA-3** 전 탭 상단 `PageHead` 도장으로 화면 각인. **IA-4** 감정서 = 서장(목차+인사) → 장별 장면(세로 기둥·비대칭). 하단 탭 5종: 月 홈·日 오늘·曆 달력·冊 서재·人 회원.
