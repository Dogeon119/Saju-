### 현재 구조의 한계 (마이그레이션 이유)

- **공유 불가에 가깝다**: CSR 단일 파일이라 풀이 결과를 남에게 보여 줄 URL이 없고, 카톡 미리보기 카드도 안 뜬다 → 사주 서비스의 성장 엔진(공유·바이럴)이 막혀 있다
- **저장 불가**: 회원·히스토리 없음. 매번 생일 재입력
- **콘텐츠와 코드가 한 파일**: 풀이 텍스트 수정에 코드 배포가 필요
- **검색 유입 0**: SSR이 없어 "정미일주 연애" 같은 검색에 안 잡힌다

## 📐 설계 원칙

1. **엔진은 순수 함수** — `lib/engine/`은 React·DB를 모른다. 프론트 미리보기와 서버 저장이 같은 코드를 쓴다
2. **콘텐츠와 코드 분리** — 풀이 텍스트 수정은 `content/`만 건드린다. 말투는 SAJU_CONTENT.md 가이드(다정한 존댓말 · 장마다 핵심 한 줄 · 2문장 문단)를 따른다
3. **계산은 ssaju에 위임** — 만세력·대운·십성·신살을 직접 구현하지 않는다. 2인 비교(궁합)처럼 ssaju가 없는 것만 `relations.ts`에 얹는다
4. **게스트 우선** — 로그인 없이 모든 기본 풀이 가능. 회원 가입은 "저장하고 싶을 때" 유도 (SION의 게스트 모드와 동일 철학)
5. **공유가 곧 마케팅** — 새 기능마다 "이걸 카톡으로 보내면 어떻게 보이나"를 먼저 설계한다

## 🔮 통변(通辯) 엔진 — 정확도·깊이 개편 (2026-07-23, 유저 "3배 자세·정확도")

**문제 진단**: base(무료) 풀이가 "한 장에 한 요소" 단일축 템플릿 룩업이라 누구에게나 비슷하게 읽혔다. `ssaju`는 만세력·원국·지장간·십신(정기)·대운은 정확하지만 ① 오행 세력을 지장간 없이 8글자로만 세고 ② 신강신약이 축소 억부(식상·재성 설기 무시) ③ 용신이 조후를 배제한 정적 룩업이라, 한난 편중 사주에서 오답이 났다.

**해결 = `lib/engine/tongbyeon.ts` (통변 상관분석 계층)**. ssaju의 신뢰 가능한 원시데이터 위에서 앱이 직접 재계산·상관:
- **지장간 가중 오행 세력**(여기 0.25·중기 0.4·정기 1.0) — 진짜 세력
- **정밀 신강신약**(득령·득지·득세 + 월지 12운성) — 근거를 노출
- **억부+조후 용신**(신강→설·극 / 신약→생·부 / 한난 극단이면 조후 우선 / 전왕·통관 분기) — "왜 이 용신인지" 근거 문장 생성
- **십신 구조 조합**(관인상생·식상생재·재생관·재다신약·상관견관·비겁태왕)
- **통근/투출 · 원국 합충형파 · 대운-원국 상호작용(용신/기신·합충) · 육친**

**적용**: 정통사주(`renderSaju`, 13장 유지)를 통변으로 재작성 — 第一장에 신강신약·용신·조후 뼈대, 각 장(일주·십성·재물·직업·건강·대운·연운·문답)이 여러 요소를 **엮어** 개인화. plain text ~2,500자 → **~7,500자**. `tongbyeon.test.ts`(전수 스윕·신강/신약/조후 공존·용신 유효) + 기존 회귀 통과(총 48테스트). 범위: 유저 확정대로 **정통사주 우선**, 이후 연애·궁합·연운 확산 예정. AI 심층풀이는 API 키 미설정으로 보류(무료 결정적 풀이 집중).
