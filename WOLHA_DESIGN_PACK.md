## §0. 다른 앱에 적용하는 법 (Adoption)

1. **CSS 이식** — §1의 CSS를 새 앱의 전역 스타일에 **`fonts` → `tokens` 순서**로 붙여넣는다.
   - **Tailwind v4** 기준(`@import "tailwindcss"` + CSS 변수). v4면 그대로 동작.
   - **Tailwind v3 / 순수 CSS** 면 변수를 그대로 쓴다 (`color: var(--paper)`, `background: var(--gold)` 식). 월하는 유틸리티보다 **시맨틱 클래스 + CSS 변수**로 짜여 있어 이식이 쉽다.
2. **폰트** — 표제는 **명조(Noto Serif KR)**, 본문·UI는 **시스템 고딕**. Next 앱이면 `next/font/google` 셀프호스팅이 원칙(§1.1). 비(非)Next 앱은 §1.1의 CDN 폴백을 쓰되, 그건 규율 예외임을 안다.
3. **규칙 적용** — 화면을 만들 때 §2를 체크리스트로 쓴다. 우선순위는 **색(`CO-*`) → 위계(`HI-*`) → 인쇄 문법(`PR-*`) → 카드(`CA-*`)**.
4. **무시할 것** — §2 안의 월하 앱 내부 참조(예: `PillarBoard`, `globals.css`, 라우트 `/saju`, 탭 마크, 커밋/로드맵)는 월하 전용이니 다른 앱에선 무시하고 **규칙 본문만** 가져간다.
5. **브랜드 치환** — 다른 앱이 월하가 아니라면 색 토큰(`--gold`·`--nakgwan`·오행 5색)만 그 앱 색으로 교체하면 된다. 나머지 규율(**무채색 지면 90%·의미색 절제·크기 위계·단일 카드·반AI 인쇄 문법**)은 브랜드 무관하게 그대로 유효하다.

> **월하 색 관습 주의:** 이 시스템에서 **붉은색(`--nakgwan`)은 "주의·경계·낙관 도장"** 이고 **금박(`--gold`)은 "길함·강조·행동 유도"** 다.
> 주식 앱처럼 **빨강=상승, 파랑=하락이 아니다.** 오행 5색(`--e-*`)은 데이터(사주판·오행 바·달력)에만 쓰는 별개 축이다. 포팅할 때 이 세 축을 섞지 마라.

## §1. 붙여넣기용 CSS

### 1.1 폰트 — 명조(Noto Serif KR) + 시스템 고딕

**원칙(월하):** 표제·핵심 문장·예언은 **명조**, 본문·UI 컨트롤은 **시스템 고딕**. 폰트는 이 2종으로 끝(`TY-1`).

**Next 앱 (권장 · 월하 실제 방식) — `app/layout.tsx`**

```tsx
import { Noto_Serif_KR } from "next/font/google";

const serif = Noto_Serif_KR({
  subsets: ["latin"],          // 한글은 글리프 자동 서브셋
  weight: ["400", "600"],       // 이 2웨이트로 끝
  variable: "--font-serif",
  display: "swap",
});

// <html className={serif.variable}> 로 주입 → CSS 변수 --font-serif 활성
```

고딕은 웹폰트를 싣지 않고 **시스템 스택**으로 간다(속도·격 둘 다). 아래 토큰의 `--sans` 참고.

**비-Next 앱 (CDN 폴백)** — Next가 아니라 셀프호스팅이 불가능할 때만. 월하 원본은 표제 웹폰트 CDN을 금지(DESIGN.md §2)하므로 이건 **이식 예외**로 간주한다.

```css
/* 명조 표제용 폴백 — 셀프호스팅이 불가능한 앱에서만 */
@import url('https://fonts.googleapis.com/css2?family=Noto+Serif+KR:wght@400;600&display=swap');
/* 고딕은 웹폰트 불필요 — 아래 --sans 시스템 스택 사용 */
```

### 1.2 토큰 — 색·폰트·라운딩 (`tokens.css`)

> 기본값 = **月夜(월야)** 밤하늘·감청 어두운 모드. `:root[data-theme="light"]` = **月白(월백)** 달빛 새벽.
> 토큰 **이름은 고정**, 값만 모드별로 갈아 끼운다 — 컴포넌트 CSS는 무수정으로 두 모드를 지원한다. (리브랜드 이력: DESIGN.md 부록 I)

```css
:root{
  /* 지면 (紙面) — 月夜: 감청빛 밤하늘 (기본·어두운 모드) */
  --ink-0:#0B0E1A;   /* 페이지 최심부 — html/body 배경 */
  --ink-1:#14182D;   /* 카드·그룹 표면 */
  --ink-2:#1C2140;   /* 카드 내부 요소·입력 배경 */
  --ink-3:#272D52;   /* hover·활성 표면 */

  --line-strong:#454E76; /* 강조 구분선·입력 테두리 */
  /* 금박 (金箔) — 별빛금(星金), 유일한 액센트 */
  --gold:#DEC583;      /* 제목 강조·활성 탭·주요 버튼·길함 */
  --gold-dim:#96865B;  /* 비활성 금·보조 강조·진행 바 */
  /* 먹 — 글자: 달빛에 물든 은라벤더 */
  --paper:#DEE1EF;     /* 본문·숫자 */
  --paper-dim:#9096B2; /* 캡션·보조·라벨 */
  /* 낙관 (落款) — 도장·주의 전용 */
  --nakgwan:#C24438;      /* 도장 테두리 */
  --nakgwan-text:#E0776B; /* 도장 글자·주의 문구·강조(em) */

  /* 폰트 */

  --sans:-apple-system,BlinkMacSystemFont,"Pretendard","Noto Sans KR","Malgun Gothic","Apple SD Gothic Neo",sans-serif;
  /* 라운딩 */
  --r-ctl:10px;   /* 버튼·입력·세그먼트 */
  --r-card:12px;  /* 데이터 카드 */
  --r-group:16px; /* 그룹 카드(폼·리스트) */
}

/* 月白(월백) — 밝은 모드 (달빛 스민 서늘한 새벽 종이) */

  --ink-1:#FAFBFE;
  --ink-2:#EFF2F9;
  --ink-3:#E2E7F2;

html{background:var(--ink-0); scroll-behavior:smooth;}
```

### 1.3 유동 스케일 컨테이너 (잘림 방지)

```css
/* 크기는 clamp()가 화면 폭을 따라 연속 조정 — 640px 절벽 금지. 좌우는 노치 safe-area와 클램프 중 큰 쪽 */
.wrap{
  max-width:760px; margin:0 auto; box-sizing:border-box;
  padding:clamp(20px,5vw,48px)
    max(clamp(16px,3vw,24px), env(safe-area-inset-right))
    clamp(96px,14vw,120px)
    max(clamp(16px,3vw,24px), env(safe-area-inset-left));
}
```

핵심 4원칙: ① 폭은 `%·fr·max-width` (고정 `w-96` 금지) ② 모바일 퍼스트, 미디어쿼리는 **확장 방향**으로만 ③ 세로 높이는 `100dvh`(`100vh` 금지) ④ Safe Area: 하단 탭 `env(safe-area-inset-bottom)` + 본문 하단 `calc(84px + inset)`.

## §2. 디자인 규칙 (DESIGN.md 전문 + 이식 거버넌스)

> 모든 강제 규칙엔 규칙 ID(`CO-1` 등)가 있고, 리뷰·논의에서 섹션 번호 대신 ID로 인용한다.
> 우선순위 충돌 시: **시각은 이 문서, 언어(풀이 문체)는 `SAJU_CONTENT.md`, 화자 톤은 `PERSONA.md`가 이긴다.**

### Part 0 — 브랜드 한 줄 & 반(反)AI 원칙

> **월하 = "감청빛 밤하늘 아래, 별빛 금으로 눌러 쓴 달빛의 감정서(鑑定書)."**

고급스러움은 색의 수가 아니라 **절제**에서 나온다. 월하의 화면은 명품 인쇄물의 문법을 따른다 — 낮은 채도, 어두운/미색 명도 베이스, 금속성 액센트 한 색, 비대칭 조판, 각이 살아 있는 도장.

**반AI 원칙:** 이 시스템의 존재 이유는 "AI가 만든 티"의 제거다. AI풍의 정체는 색이 아니라 **문법**이다 — 가운데 정렬 + 알약 뱃지 + 균등 라운딩 + 균등 패딩 + 그라데이션. 이 조합을 금지하고 인쇄물 문법(비대칭·크기 위계·세로쓰기·괘선·도장)으로 대체한다. → 강제 규칙은 `PR-*`.

### Part 1 — 원칙 (Principles)

#### 1.1 제품 정체성 — 3초 룰

| ID | 규칙 |
|---|---|
| **ID-1** | 첫 화면을 3초 본 사용자가 위 브랜드 한 줄("달빛 아래의 감정서")을 느껴야 한다. |
| **ID-2** | 화면의 주인공은 **감정(鑑定)의 언어** — 명조 표제 한 줄, 또는 사주판/점수/장면 제목이다. 위젯 나열이 아니다. |
| **ID-3** | 대시보드(통계 나열)가 아니라 **읽는 화면(감정서)** 이다. 숫자보다 문장이, 문장보다 여백이 격을 만든다. |
| **ID-4** | 게스트가 홈에서 알고 싶은 순서: **① 여기가 무엇을 봐주는 곳인가 → ② 어떤 감정을 받을 수 있나(모드) → ③ 지금 바로 시작.** |

**판단 기준 (ID-5):** *"이 요소가 위 3초 메시지를 강화하는가? 아니면 그냥 '있으면 좋은 정보'인가?"* 후자면 홈에서 뺀다(다른 탭 또는 스크롤 하단).

#### 1.2 제1원칙 — 위계 (Visual Hierarchy)

모든 화면은 **3단 위계**를 가진다. 같은 크기·같은 여백의 요소를 나열하는 것을 금지한다.
월하의 위계는 **그림자·브랜드 배경으로 만들지 않는다 — 크기·명조·명도·괘선으로만** 만든다(→ `PR-6`, `HI-4`).

| 단계 | 역할 | 개수 | 시각적 처리 (월하식) |
|---|---|---|---|
| **HERO** | 그 화면의 단 하나의 주인공 | **1개** | 가장 큼 · **명조 대형**(장면 제목 28px / 점수 56px / 히어로 문장) · 여백 넉넉 · 부분 강조만 `--gold` |
| **PRIMARY** | 주인공을 보조하는 핵심 | 2~4개 | 중간 · 핵심 문장(`.key`, 명조 19px + 상하 괘선) · 스탯 행 · 사주판/오행 바 |
| **TERTIARY** | 참고 정보 | 나머지 | 작음 · `--paper-dim` 저대비 · 캡션·괘선 주석·배지 · 스크롤 아래 |

| ID | 규칙 |
|---|---|
| **HI-1** | 화면마다 HERO는 **정확히 1개**. 강조를 여러 개 하면 강조가 사라진다. "이 화면에서 제일 중요한 것 하나"를 정하고 나머지는 의도적으로 죽인다. |
| **HI-2** | 한 화면(첫 스크롤)에 보이는 카드·블록은 **최대 5~8개**. 그 이상은 스크롤로 내린다. |
| **HI-3** | HERO는 여백을 넉넉히 차지해도 된다(홈 히어로 문장 + 수묵 달 장식은 상단 큰 블록). |
| **HI-4** | 위계는 **크기·명조·괘선**으로 만든다. 굵기만으로, 또는 색·그림자·브랜드 배경으로 위계를 만들려 하지 않는다. |

### Part 2 — 파운데이션 (Foundations)

#### 2.1 색 시스템 — "색은 의미가 있을 때만" ★가장 중요

> 원칙 (**CO-1**): **화면의 90%는 무채(ink) 명도 계열(月夜 감청 밤하늘 / 月白 달빛 새벽)이다. 색은 ① 금박(브랜드·길함) ② 오행(데이터 의미) ③ 낙관(주의·경계), 이 셋에만 쓴다.** 장식으로 쓰는 색은 전부 제거한다.

**2.1.1 팔레트 — CO-2: §1.2 토큰에 없는 색은 UI에 쓰지 않는다.** (검증: `GV-1`)

**세 개의 의미 축 — 서로 유용(流用) 금지:**

| 축 | 토큰 | 의미 | 절제 규칙 |
|---|---|---|---|
| **금박** | `--gold` / `--gold-dim` | 브랜드·강조·**길함**·행동 유도·활성 표시 | 한 화면에서 제목 강조 + 활성 표시 정도. **본문 글자에 금색 금지.** |
| **낙관** | `--nakgwan` / `--nakgwan-text` | 도장·**주의·경계**·경고 문구·강조(em) | **화면당 도장 1개 이하.** 붉은색이 흔해지는 순간 격이 죽는다. |
| **오행 5색** | `--e-mok/hwa/to/geum/su` | 사주판·오행 바·달력의 **데이터** | UI 액센트로 유용 금지. 목木·화火·토土·금金·수水 고정 매핑. |

| ID | 규칙 |
|---|---|
| **CO-1** | 화면의 90%는 무채(ink) 명도. 색은 금박·오행·낙관 세 축에만. |
| **CO-2** | 팔레트(§1.2)에 없는 색은 UI에 쓰지 않는다. 순수 흰색(#FFF)·순수 검정(#000)도 금지 — 月白 지면은 전부 서늘한 미색(최명 #FAFBFE), 月夜는 전부 밤빛(최암 #0B0E1A). |
| **CO-3** | `--gold`는 최중요 강조·길함·행동 유도에만. 카드 테두리·아이콘 배경 장식에 남발 금지. |
| **CO-4** | 붉은색(`--nakgwan`)은 **주의·경계·도장**에만. 길함·강조에 붉은색 금지(그건 금박). 오행 火(`--e-hwa`)와 혼동 말 것 — 화火는 데이터 전용. |
| **CO-5** | 오행 5색은 사주판·오행 바·달력 마킹 전용. UI 액센트 금지. |
| **CO-6** | 한 화면에 의미색(금·낙관·오행)이 과하면 줄인다. **그라데이션(2색 이상)·네온·보라 계열 전면 금지.** |

**2.1.2 달력·길흉 의미 매핑 (월하 고유 데이터 축):** 길일 = `--gold` · 주의일(충·원진·해) = `--nakgwan-text` · 절기 = `--e-mok` · 손없는날 = `--e-su`. 이 매핑은 "달력/일진" 데이터에서만 쓰고, 일반 UI 강조로 새어 나가지 않는다.

**2.1.3 제거 대상 (색 부채):** 앰버/노랑 계열 → 금박 또는 텍스트로 · 보라 `#8B5CF6` → 제거 또는 금박 · 스카이블루/틸/네온 → 전부 금박/오행/중립 · 카드마다 다른 색 테두리·색 아이콘 배경 → 중립 `--ink-2` + `--paper-dim`.

#### 2.2 타이포 (Typography)

| ID | 규칙 |
|---|---|
| **TY-1** | 폰트는 **명조·고딕 2종**으로 끝. 제3의 폰트 금지. |
| **TY-2** | 숫자·수치는 `font-variant-numeric: tabular-nums` (자리 흔들림 방지). |
| **TY-3** | 위계는 **크기 대비**로 만든다(→ `HI-4`). "핵심 한 줄 —", "💡 TIP", "⚠️" 같은 **라벨식 위계 금지** — 라벨 대신 크기·괘선으로 격을 만든다. |
| **TY-4** | 최소 크기 **12px**(초미세 축 라벨 전용), 실질 최소 **13px**. 본문 행간 **1.9 이상** 유지 — 좁히지 않는다. |

| 역할 | 폰트 · 크기/굵기 |
|---|---|
| 히어로 문장 / 장면 제목 | 명조 · clamp(23~46) / 600 |
| 점수(대형 수치) | 명조 · 56 / 600, `tabular-nums` |
| 페이지 타이틀 | 명조 · clamp(20~27) / 600 |
| 핵심 문장(`.key`) · 예언 | 명조 · 19 / 600, 상하 괘선 |
| 본문 | 고딕(또는 명조 인용) · 17 / 1.95 |
| 라벨·캡션 | 고딕 · 13~13.5 / 600, `--paper-dim` |

> 편집 디자인 문법: **명조는 표제·핵심 문장·예언·서장 인사**에, **고딕은 본문·UI 컨트롤**에. 둘을 섞어 격을 만든다.

#### 2.3 여백 & 그리드 (Spacing)

> "디자인보다 여백이 더 많아야 한다." 답답함의 원인은 대부분 여백 부족이다. **여백은 낭비가 아니라 위계 도구다.**

| ID | 규칙 |
|---|---|
| **SP-1** | **8pt 그리드** — 여백·패딩은 8의 배수(미세 보정 ±2 허용). 간격 스케일: 4·8·12·16·24·32·48. |
| **SP-2** | 카드 내부 패딩: `clamp(16px,2.6vw,24px)`. |
| **SP-3** | 섹션 사이 24~32. 위계가 바뀌는 지점(HERO→PRIMARY)엔 32 이상 + 괘선 1개. |
| **SP-4** | 페이지 컨테이너 `max-width:760px`, 유동 패딩(§1.3). 고정 폭 금지. |

### Part 3 — 패턴 (Patterns)

#### 3.1 인쇄물 문법 (반AI) — 월하 고유 강제 규칙

| ID | 규칙 |
|---|---|
| **PR-1** | **알약(pill) 뱃지 · `border-radius:9999px` · `rounded-full` 전면 금지.** 상태·표식은 **낙관 도장**(각진 사각) 또는 사각 괘선 칩으로. |
| **PR-2** | **낙관 도장** — 한자 1자, 34px, `border:1.5px solid var(--nakgwan)`, **라운딩 0**, 글자 `--nakgwan-text` 명조. 緣財戒吉命情 등 문맥 한자. **화면당 1개 이하**(→ `CO-4`). |
| **PR-3** | **세로쓰기 챕터 기둥** — 장 표기는 본문 좌측 세로 기둥(`writing-mode:vertical-rl; letter-spacing:7px; --paper-dim; 명조`). 가운데 정렬 타이틀 금지. 장 번호는 한자 숫자(一二三…). |
| **PR-4** | **가운데 정렬은 도장·인장 요소 내부에만.** 본문·제목·카드 정렬은 좌측 기준 비대칭 조판. |
| **PR-5** | **이모지 금지.** 아이콘이 필요하면 한자 도장 또는 텍스트로. |
| **PR-6** | **`box-shadow` 금지**(0 제외) · **blur 장식 금지**. 표면 분리는 **헤어라인 보더 + 지면 명도차**(ink-0 페이지 위 ink-1 카드)로. (예외: 상단 sticky 탭·하단 탭의 backdrop blur는 허용.) |
| **PR-7** | **라운딩 규율** — 카드 12~16 · 버튼/입력/세그먼트 10 · **도장·괘선·사주판 글자 칸·구분선은 0**. 그 사이 임의 값 금지. |
| **PR-8** | **세그먼트 컨트롤**로 2~3지선다(양력/음력·성별) — pill 아님, 라운딩 10 사각. 트랙 `--ink-2`, 선택 썸 `--ink-3` + `--paper`. |

#### 3.2 카드 규율 (Card Discipline)

카드는 "묶어야 할 이유가 있을 때"만 만든다. 지표 하나에 카드 하나 = 카드 인플레이션.

| ID | 규칙 |
|---|---|
| **CA-1** | 단일 숫자 지표 여러 개를 각각 카드로 만들지 않는다 → **하나의 스탯 행(row)** 으로(괘선/여백으로 나눔). 예: `감정서 수 · 입회일 · 저장 궁합`은 카드 3개가 아니라 한 블록의 사각 스탯 칩 행. |
| **CA-2** | **카드는 한 종류 룩만 존재한다** — 라운딩·보더·패딩을 카드마다 다르게 쓰지 않는다. 유일한 카드 룩: `background:var(--ink-1); border:1px solid var(--line); border-radius:var(--r-group); padding:clamp(16,2.6vw,24)` (그림자 없음, → `PR-6`). |
| **CA-3** | **장면 카드 3유형만:** `data`(사주판·오행 바 — `--ink-1` 카드) · `story`(본문 — 카드로 감싸지 않고 지면에 직접 비대칭 조판) · `warn`(주의·당부 — 괘선 상단 + 문구 + 낙관 도장). |

#### 3.3 빈 상태 (Empty State) & 장식 일러스트

| ID | 규칙 |
|---|---|
| **EM-1** | 데이터가 없으면 **큰 빈 UI(빈 차트·빈 표)를 그리지 않는다.** 대신 낮은 높이의 수묵 일러스트(`.empty-art`) + 짧은 안내 + (있으면) 행동 버튼 1개. |
| **EM-2** | 데이터가 생기면 그때 원래 크기로 보여준다. |
| **EM-3** | **장식 일러스트는 수묵 선화(線畵) 문법** — 달·구름(운문)·산 능선·별. 인라인 SVG만(`components/art.tsx`), 색은 토큰 `var()`만(hex 금지), `aria-hidden` + `pointer-events:none`, 본문 뒤(z 0). **화면당 주 장식 1개**(히어로급). 사진·래스터·그라데이션 채색 금지. |

#### 3.4 모션 (Motion)

움직임은 **지면에 스며드는 잉크**처럼 — 짧고, 낮고, 한 방향(아래→위)으로만.

| ID | 규칙 |
|---|---|
| **MO-1** | 진입은 `fadeUp`(8~14px 상승 + 페이드) **단일 어휘**, 0.25~0.5s, `cubic-bezier(.22,1,.36,1)`. 리스트·그리드는 `.stagger`로 자식당 50ms 지연. |
| **MO-2** | 버튼류 `:active`에 `scale(.98)` press 피드백. 차트 막대는 `scaleX/scaleY` 성장(수치 자체는 애니메이션 안 함). 모달=딤 fadeIn + 본체 riseIn, 토스트=하단 슬라이드업 후 자동 소멸. |
| **MO-3** | **금지:** bounce·spring·회전 장식, 0.6s 초과 진입, hover 큰 위치 이동, 무한 반복(상태 점 pulse만 예외). `prefers-reduced-motion:reduce`에서 **전부 무효** — 기능은 애니메이션 없이 완결. |

| ID | 규칙 |
|---|---|
| **IA-1** | 월하 홈은 **감정 도구 허브**다(SION의 "AI 판단 hero"와 다름). 통계·대시보드 위젯을 홈에 쌓지 않는다(→ `ID-5`) — 참고 지표는 각 전용 탭(오늘·달력·서재)이 주인. |
| **IA-2** | 모드는 **행이 곧 상품** — 도장(명조 한자) + 제목 + 한 줄 설명. 데스크톱은 행 리스트, 모바일은 기능 전체가 첫 화면에 들어오는 **2열 가로 타일**(설명·셰브런 숨김). |

### Part 5 — 거버넌스 (Governance)

#### 5.1 커밋(적용) 전 체크리스트

1. 이 색은 팔레트(§1.2)에 있는가? 순백/순검을 안 썼는가? (**CO-2**)
2. 이 색은 금박·오행·낙관 셋 중 하나인가? 아니면 무채로. 세 축을 섞지 않았는가? (**CO-1, CO-4, CO-5**)
3. 그라데이션·네온·보라·box-shadow·blur 장식을 안 썼는가? (**CO-6, PR-6**)
4. 이 화면의 HERO는 하나인가? 첫 스크롤 블록이 8개 이하인가? 위계를 크기·명조·괘선으로 만들었는가? (**HI-1, HI-2, HI-4**)
5. pill·이모지·가운데 정렬 남발·라벨식 위계를 피했는가? 도장은 화면당 1개 이하인가? (**PR-1, PR-5, PR-4, TY-3, CO-4**)
6. 단일 지표를 카드로 쪼개지 않고 스탯 행으로 묶었는가? 카드 룩이 한 종류인가? (**CA-1, CA-2**)
7. 빈 데이터에 큰 빈 UI를 그리지 않았는가? (**EM-1**)
8. 라운딩·8pt 그리드·유동 스케일(clamp·dvh·safe-area)을 지켰는가? (**PR-7, SP-1, §1.3**)

#### 5.3 Do / Don't

*Do* — 화면마다 HERO 하나를 먼저 정한다(HI-1) · 위계는 크기·명조·괘선으로(HI-4) · 색 쓰기 전 "금박·오행·낙관 중 뭔가"를 자문(CO-1) · 지표는 스탯 행으로(CA-1) · 빈 데이터는 수묵 일러스트 + 짧은 안내로(EM-1) · pill 대신 낙관 도장(PR-1·PR-2).

*Don't* — 같은 크기 카드 8개 이상 나열(HI-2) · 그림자·그라데이션·blur 장식(PR-6, CO-6) · pill·이모지·가운데 정렬 남발(PR-1, PR-5, PR-4) · 붉은색을 길함/강조에(CO-4) · 오행색을 UI 액센트로(CO-5) · 홈에 참고 지표 대시보드 쌓기(ID-5, IA-1) · 라벨식 위계("핵심 한 줄 —", "TIP")(TY-3).

#### 5.4 예외 구역 — 관리자 콘솔

`/admin`(`.admin-shell` 하위)은 **브랜드 규율의 예외 구역**이다 — 내부 도구는 모던 화이트 SaaS 문법(흰 지면 · 인디고 단일 액센트 · 시스템 고딕 · 미세 그림자 허용)을 따른다. 본문 §1 토큰·금박·낙관·명조를 쓰지 않는다. 이모지 금지(PR-5)는 이 구역에도 유지. `GV-1` hex 검증은 이 구역을 제외한다.
